# 照片目录

将你的照片文件放在这个目录下，构建时会自动处理。

## 📁 目录结构示例

```
photos/
├── 2024/
│   ├── 春天/
│   │   ├── IMG_001.jpg
│   │   └── IMG_002.heic
│   ├── 夏天/
│   │   └── photo.jpg
│   └── 秋天/
│       └── landscape.jpg
└── 2023/
    ├── 旅行/
    │   └── trip.jpg
    └── 日常/
        └── daily.jpg
```

## 📸 支持的格式

- **JPG / JPEG** - 标准照片格式
- **PNG** - 支持透明背景
- **HEIC** - Apple 设备照片格式（会自动转换）
- **TIFF** - 专业摄影格式（会自动转换）
- **Live Photos** - iPhone 动态照片（自动检测）

## 🏷️ 文件命名建议

- 使用有意义的文件名，如 `sunset-beach.jpg`
- 避免特殊字符，推荐使用 `-` 或 `_`
- 支持中文文件名

## 📋 EXIF 信息

照片中的 EXIF 信息会被自动提取并展示，包括：

- 📷 相机型号
- 🔭 焦距
- ⚙️ 光圈和快门
- 📍 GPS 位置（如果有）
- 📅 拍摄时间

确保照片包含 EXIF 信息以获得最佳展示效果。

## 🚀 使用流程

1. 将照片放入此目录
2. 运行 `pnpm build` 构建站点
3. 照片会自动处理并生成缩略图
4. manifest.json 会记录所有照片信息

## 💡 提示

- 大量照片首次构建会比较慢，请耐心等待
- 后续增量构建只处理新增/修改的照片
- 建议按年份或主题组织照片目录
